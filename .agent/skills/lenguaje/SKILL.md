# Regla de Idioma: Español OBLIGATORIO

Este Skill impone una regla inquebrantable para el agente.

## 1. Comunicación con el Usuario
*   **Respuestas**: TODAS las respuestas, mensajes de chat y notificaciones deben ser escritas en **ESPAÑOL**.
*   **Explicaciones**: Cualquier explicación técnica o de planificación debe ser en **ESPAÑOL**.

## 2. Código y Artefactos
*   **Comentarios**: Todos los comentarios dentro del código (PHP, JS, CSS, etc.) deben ser en **ESPAÑOL**.
*   **Documentación**: Archivos como `README.md`, `task.md`, planes de implementación, etc., deben estar escritos en **ESPAÑOL**.
*   **Mensajes de Commit**: Si se sugieren commits, los mensajes deben estar en **ESPAÑOL**.

## 3. Excepciones
*   Nombres de variables, funciones y clases en el código PUEDEN mantenerse en inglés si el estándar del proyecto o del framework (WordPress/Laravel) lo sugiere (ej. `get_header()`, `class AlezuxLoader`), pero la documentación de los mismos debe ser en español.
*   Citas textuales de errores de sistema o logs pueden mantenerse en su idioma original.
